## Appendices

### A. Reference Models for Performance Testing

1. **Simple Assembly** (baseline)
   - 5-10 parts
   - Basic primitives + booleans
   - Target: < 500ms evaluation

2. **Medium Assembly** (typical use case)
   - 20-50 parts
   - Mix of primitives, features, assemblies
   - Target: < 1.0s evaluation

3. **Complex Assembly** (stress test)
   - 100+ parts
   - Advanced features (fillets, patterns, etc.)
   - Target: < 1.5s evaluation (P95)

### B. Generated Node Categories

1. **Primitives** (7 nodes)
   - Box, Sphere, Cylinder, Cone, Torus, Wedge, Prism

2. **Booleans** (4 nodes)
   - Union, Intersection, Difference, Cut

3. **Features** (15+ nodes)
   - Fillet, Chamfer, Shell, Draft, Offset, Thicken, etc.

4. **Transformations** (8 nodes)
